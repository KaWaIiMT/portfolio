# 部署到 GitHub Pages — 操作步骤

## 准备工作已就绪

静态文件已构建完成（`out/` 目录下有 6 个路由：`/`, `/about`, `/projects/*`, `/404`）。

---

## 步骤 1：登录 GitHub CLI

打开 PowerShell，运行：

```powershell
gh auth login
```

按提示选择：
- `GitHub.com`
- `HTTPS`
- `Login with a web browser`

复制终端里显示的 8 位验证码，浏览器会自动弹出，粘贴确认即可。

---

## 步骤 2：创建仓库 & 推送

```powershell
cd "D:\_College\保研\作品集"

# 创建 GitHub 仓库（名字叫 个人作品集 不太合适，英文名更 URL-friendly）
gh repo create portfolio --public --source=. --remote=origin --push
```

如果上面命令报错说已有 git 仓库，就用：

```powershell
git remote add origin https://github.com/你的用户名/portfolio.git
git push -u origin master
```

---

## 步骤 3：启用 GitHub Pages + 自定义域名

### 方式 A — 使用 gh CLI：
```powershell
gh api repos/你的用户名/portfolio/pages -X POST -f "source[branch]=master" -f "source[path]=/out"
```

但 GitHub Pages 通常要求 `/`（根目录）或 `/docs`。`/out` 不是标准路径。

### 方式 B（推荐）— 用 `.github/workflows/deploy.yml` 自动部署：

我帮你创建一个 GitHub Actions 工作流文件，它会自动把 `out/` 发布到 `gh-pages` 分支：

---

## 步骤 4：创建自动部署 Action

在项目根目录创建 `.github/workflows/deploy.yml`，内容如下：

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [master]

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
      - run: npm ci
      - run: npm run build
      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./out
      - id: deployment
        uses: actions/deploy-pages@v3
```

推到 GitHub 后：

1. 打开 `https://github.com/你的用户名/portfolio/settings/pages`
2. **Source** 选 `GitHub Actions`
3. 等 Action 跑完，页面地址会显示在 Settings → Pages 顶部

---

## 步骤 5：访问你的网站

部署完成后，网址是：

```
https://你的用户名.github.io/portfolio
```

等 1-2 分钟 GitHub Pages 首次构建完成后就能访问了。

---

## 如果要自定义域名

在 `public/` 目录下放一个 `CNAME` 文件，内容写你的域名，然后去域名 DNS 里加一条 CNAME 记录指向 `你的用户名.github.io`。
